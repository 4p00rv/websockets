Overview

The Binary.com Websockets API uses the state-of-the-art WebSockets protocol to allow fast, two-way messaging between client programs and Binary.com trading services.

To open a connection to Binary.com WebSocket service (in Javascript), do:

var ws = new WebSocket('wss://ws.binaryws.com/websockets/v3');

For other languages, View code samples

You can now send data up and down the websocket using any of the messages described in this Reference.

Note: to receive responses where English text has been translated to another language, append the language code to the connection URL as follows:

var ws = new WebSocket('wss://ws.binaryws.com/websockets/v3?l=RU');

Data transfer in both directions is done with structured JSON-encoded data. All field name identifiers, and most field values, will be double-quoted in the encoded data. The same applies to receiving messages: each message is a serialized JSON-encoded string.

API Functionality

Authentication

The API provides certain functions that require authentication (e.g. portfolio) whilst other functions do not require authentication (e.g. ticks). Authentication is provided by way of API tokens, which binary.com users can generate by logging into their accounts and visiting https://www.binary.com/user/api_token. An Oauth service is being developed and will be available in early 2016. With a user's API token, you may call authorize.

Streams

Certain functions generate streams (e.g. ticks) whilst other functions do not. Please use forget to cancel an outstanding stream.

Binary.com Account Opening

To open binary.com accounts via the API, start by verifying the user's email address using verify_email. You may then call new_account_virtual to open a virtual-money account. Note: you may top-up virtual-money funds using topup_virtual. To open a real-money account, you may use residence_list and states_list to retreive data to create the account opening form. Once you have the required data from the user, call new_account_real. If you are a registered affiliate, remember to pass in the affiliate_token parameter, so that you may earn affiliate commissions for the new account. You can determine which Landing Company (i.e counter-party) the account will be opened with, based on the user's jurisdiction, using landing_company, and you can find out details about that landing company using landing_company_details.

Accounting functionality

You may generate a user's portfolio, statement, and profit_table, as well as retrieve his account balance.

Option prices

To price an option, call proposal. To price options which are open in a user's portfolio, call proposal_open_contract.

Payment agents

Clients may withdraw to a payment agent using paymentagent_withdraw and payment agents may credit a client account using paymentagent_transfer.

Utility calls

If need be, use ping and server time.

Open-Source Projects

Binary.com maintains a number of open-source projects on github. Feel free to fork these repositories, and if you wish to submit an enhancement to a project, submit a pull-request. You may use desktop.github.com if you are unfamiliar with git.

The source for this API documentation website itself is maintained at https://github.com/binary-com/websockets.

Binary.com is developing an open-source mobile application at https://github.com/binary-com/mobile as well as a next-generation user-interface which acts as both a mobile application and a desktop application at https://github.com/binary-com/binary-next-gen. A separate project is underway to build a desktop application at https://github.com/binary-com/webtrader.

The front-end code for the binary.com website itself is open-source at https://github.com/binary-com/binary-static.

Binary.com's open-source projects are typically published via the Github Pages facility.
